Google Analytics

После установки аккаунта нужно связать его с аккаунтом Google Webmasters Tools. Появятся три новых отчета: Источники трафика > Поисковая оптимизация > Запросы / Страницы входа / Сводка географических данных.

См. также введение в GA.

Отчеты по поисковой оптимизации

С помощью этих отчетов можно:

  • Определить ключевые слова с низким CTR, но хорошей средней позицией и заменить Title и мета-тег description, чтобы повысить CTR.
  • Определить страницы входа с хорошим CTR, но низкой средней позицией, чтобы поправить оптимизацию на уровне страницы.
  • Получить более точные данные о географии посетителей.

Цели для разных сайтов

Затем следует установить цели. Они зависят от главных задач сайта: продажа товара или услуги, получение прибыли от рекламы и т.п.

  • Интернет-магазин — надо включить Конверсии > Электронная торговля
  • Продажи/загрузки — если что-то продается не в интернет-магазине (например, книга), можно поставить цель на страницу скачивания
  • Привлечение посетителей — люди, которые проводят на сайте больше минуты
  • Читатели — люди, которые просматривают минимум две страницы сайта.
  • CTA — используется отслеживание действий.
  • Лучшие баннер — также используется отслеживания действий
  • Подписчики email-рассылки.

Стандартная последовательность просмотра страниц сайта

Можно выбрать исходное условие (источник траффика, ключевое слово, страну посетителя) и проследить дальнейшие переходы и страницу выхода.

  • Аудитория > Поток посетителей
  • Конверсии > Цели > Переход к цели

Отслеживание переходов по ссылкам из различных рекламных материалов

Источники трафика > Источники > Кампании. Чтобы отследить переходы по ссылкам из рекламных материалов, надо пометить соответствующие ссылки с помощью параметров в «Компоновщике URL» Google (затем ссылки можно сократить в goo.gl и т.п.). Ссылки из рекламы AdWords метятся автоматически, если аккаунты связаны (https://adwords.google.ru > инструменты и анализ > Google Analytics > нужный аккаунт).

Многоканальные последовательности

Конверсии > Многоканальные последовательности. Здесь сравниваются пути пользователя к конверсии: прямой трафик, поиск, баннеры, email-рассылка, соц. сети. взаимодействие различных источников трафика приводит к конверсиям. Конверсией может быть покупка в интернет-магазине или любое другое действие, которое определена как цель.

Отслеживание действий

Отслеживание событий позволяет получать такие данные, как:

  • Сколько людей скачивают книгу
  • Какие баннеры / объявления работают лучше.
  • Каккие люди регистрируются.
  • Кто пользуется кнопками просмотра видео.
  • Какие ошбки мешают пользователям в процессе расчета.

Добавить в атрибут onclick ссылок или кнопок отправки формы (button, input) функциию_gaq.push с тремя обязательными параметрами _trackEvent, category, action и двумя необязательными параметрами opt_label, opt_value.

onclick="_gaq.push(['_trackEvent', 'category', 'action', 'opt_label', 'opt_value']);"

  • Category используется для обозначения типа отслеживаемого события скачивание электронной книги (ebook), просмотр видео (video), форма регистрации (signup), реклама (ad) и т.п.
  • Action — для обозначения типа действия — клик, button (форма), play, stop — или для обозначения области страницы, где расположена ссылка или кнопка.
  • Label — для уточнения.
  • Value — для оценки действия, которое потом подсчитывается в статистике достижения целей.

Например:
<a href="http://www.webanalyticsblueprint.com/" onclick="_gaq.push(['_trackEvent', 'Ads', 'Sidebar', 'WAB']);">

Ads — тип ссылки, Sidebar — область страницы, WAB — уточнение типа действия (Windows Address Book).

После того, как onclick добавлен во все нужные ссылки/кнопки, нужно настроить цель в GA: все аккаунты (выпадающее меню в левом верхнем меню) > [Нужный сайт] > Администратор (справа вверху) > вкладка во втором ряду «Цели» > +Цель (в одном из трех наборов; или изменить существующую).

Отслеживание поиска по сайту

Стандартные отчеты > Содержание > Поиск по сайту. Помогают более точно определить что пользователи ищут на сайте. Для торговли полезно в плане расширения ассортимента. Для блога — в плане расширения тематики.

Настройка

  1. Все аккаунты (выпадающее меню в левом верхнем меню) > [Нужный сайт] > Администратор (справа вверху) > вкладка во втором ряду «Настройки профиля» > флаг «Отслеживать поиск по сайту».
  2. В поле «Параметр запроса» ввести параметр URL, который использует CMS для поиска для сайта. Чтобы обнаружить этот параметр достаточно поискать что-нибудь и в URL результатав поиска посмотреть, какое слово или буква стоит перед поисковым запросом (знак равенства опускается). Например, для пользовательского поиска Google такимп араметром будет q.
  3. Флаг на «Удалять параметры запроса из URL».
  4. Если используется фильтры поиска (выпадающие меню) в «Категории поиска по сайту» нужно указать соответствующие параметры URL.

До 20 сводок

Моно объединять информацию по содкам. Home > Сводки > + Добавить сводку. Отчеты в сводку можно добавлять, как непосредственно из самой сводки (не очень удобно), так и из стандартных отчетов.

Статистика в реальном времени

Home (слева в верхнем меню) > В реальном времени. Можно посмотреть географию посетителей, откуда они пришли, и что сейчас смотрят.

Скорость загрузки страниц

Содержание > Скорость загрузки сайта. Здесь можно посмотреть, какие страницы следует облегчить — параметры «Показатель отказов» и «Процент выходов»

Вкладка «Эффективность» — процент загруженных данных по периодам: в первую секунду посещения, в первые три и т.д.

«Наложение данных на карту» — скорость загрузки по странам.
