SEO

См. также интернет-маркетинг, онлайн-маркетинг: цикличность процесса, интернет-маркетинг нового продукта, самореклама, маркетинг на основе БД и шаблоны маркетингового плана: в электронных таблицах Google и в Basecamp.

Целями интернет-маркетинга является увеличение трафика (количества посетителей) веб-сайта и, в конечном итоге, привлечение новых клиентов. Задача решается разными способами: покупкой трафика, рекламой, оптимизацией под описковые запросы — Search Engine Optimization.

SEO является наиболее экономичным (но все равно, не дешевым) иснтрументом интернет-маркетинга: чем выше позиция сайта в результатах поиска по определенным словам и выражениям, тем больше его аудитория. Процесс этот бессрочный и должен поддерживаться постоянно, а начинаться — с трехэтапной подготовки:

  • Аналитический этап.
  • Повышение релевантности (англ. relevant) — степень соответствия запроса и найденного. сайта интересующим поисковым запросам.
  • Наращивание «ссылочной массы» — этот этап больше, чем два предыдущих влияет на успех кампании. Кроме SEO-выгод входящие ссылки приносят и прямую пользу — прямые переходы посетителей.

Шпаргалка Факторы, влияющие на положение страницы в результатах поиска

Аналитический этап

  1. SWOT-анализ:
    • Strength (Сильные стороны)
    • Weaknesses (Слабые стороны)
    • Opportunities (Возможности)
    • Threats (Угрозы)

    Главный показатель для SWOT-анализа в области SEO: текущие позициии в результатах поиска. Они вычисляются с помощью специальных служб. Задача оптимизитарора максимально улучшить эти показатели: в идеале — вывести в первую десятку по согласованным поисковым запросам.

    Прочие качественные характеристики ресурса (посетители, источники трафика, ссылки, поисковые запросы etc) получаем с помощью специальных онлайн-служб:

    • Google Analytics
    • Инструменты для веб-мастеров Google (заодно устанавливаем географическое таргетирование)
    • Статистики поиска Google
    • Похожих запросов Google
    • Яндекс.Вебмастер
    • Bing Webmaster Tools
    • Marketing Grader

    Желательно в этих службах сделать снимки экранов, чтобы запротоколировать исходные позиции.

  2. Составление семантического ядра Иными словами, набора слов, которые предположительно могут использоваться для поиска сайта.. Задача решается с помощью:
    • Статистики по поисковым запросам Google
    • Оценщика трафика Google
    • Статистики Яндекса
    • Seodigger: смотрим по каким запросам находят конкурентов — заполняем графу URL, выбираем Google 11M Russian Base (русскоязычные запросы в Google), ищем.
    Стратегия продвижения может быть ориентирована либо на большое количество редких поисковых запросов (низкочастотные запросы — НЧ; «длинный хвост»), либо на модификации самого популярного релевантного запроса (СМРЗ). Например:
    • СМРЗ + редкое для ниши, но в целом распространенное фирменое слово сайта (ФСС). Например, PR-агентство цитаты". Нужно добиваться, примерно, 20-процентного результата от общей ссылочной массы.
    • Популярная модицикация СМРЗ + ФСС. Например, "московское PR-агентство цитаты" — 15%.
    • СМРЗ — 10%.
    • Остальное — популярные модификации СМРЗ, по 5%.
    Процесс разработки семантического ядра, как любой маркетинговый процесс цикличен и должен повторяться каждый год — слова, по которым люди ищут продукты или услуги меняются, Google Analytics открывает новые возможности.
  3. Конкурентный анализ. Следует выяснить, сколько и каких сайтов ссылается на лидеров SERP (англ. Search Engine Result Position) — позиция в результатах поиска. по интересующим запросам. «Каких» означает не только тематику, но и показатели значимости интернет-ресурсов для Google и Яндекс — в первую очередь, индексы PageRank (PR) Алгоритм, по которому Google вычисляет общественную значимость страницы путем подсчета важности ссылок на нее. Корелляция с результатами поиска не прямая, но других доступных критериев оценки потенциала страницы нет. и тИЦ Алгоритм, по которому «Яндекс» вычисляет «авторитетность» интернет-ресурсов. Определяется по качественным характеристикам ссылок с других сайтов. На результаты поиска не влияет, но используется для сортировки каталога «Яндекса». (а также MozRank Рейтинг авторитетности входящих ссылок от крупнейшей SEO-компании мира SEOMoz.). Эти данные потребуются в дальнейшем при наращивании ссылочной массы: чтобы выйти в лидеры SERP, нужно привлечь массив выходящих ссылок, суммарные показатели значимости которого не уступали бы характеристикам конкурентов. В качестве отправных точек используем результаты:
    • SeoQuake (модуль Firefox, который помимо прочих полезных функций выдает в SERP такие параметры найденных сайтов и отдельных страниц, как PR, тИЦ, входящие ссылки, учитываемые Google и Яндекс и т. д.)
    • SEMRush — cервис для исследования конкурентов, позволяет узнать запросы, по которым сайты попадают в SERP или блоки контекстной рекламы
    • Link Research Tool — платный сервис просмотра входящих ссылок на любой из указанных ресурсов.
    • Keyword Difficulty Tool
    • расширенные запросы:
      1. site:example.com "маркетинговые коммуникации"
        в Google — Какая страница на example.com наиболее соответствует (релевантна) запросу «маркетинговые коммуникации» (пиар, BTL)
      2. "example.com" -site:example.com
        в Google — входящие ссылки на example.com
      3. site:example.com
        в Yahoo — поиск страницы заданного ресурса с наиболее высоким Web Rank Аналог PageRank. (не обязательно первой); в Google — поиск сайта в индексе.
      4. "пиар" linkdomain:example.com -site:example.com linkdomain:url2.com -site:url2.com -link:url3.com
        в Yahoo — сайты, на которых встречаются слово «пиар», ссылающиеся на example.com и на url2.com, но не ссылающиеся на url3.com. Источники входящих ссылок сортируются по авторитетности.
      5. link:example.com -site:example.com
        В MSN linkdomain заменяем link.
      6. intitle:пиар+BTL "маркетинговые коммуникации"
        в Google — поиск страниц по словам не только в текстах, но и в заголовках title (в данном случае — поиск страниц, в title которых встречаются ключевые слова «пиар» и «BTL», а в тексте — ключевая фраза «маркетинговые коммуникации»)
      7. "поисковый запрос" /domainlinks
        в blekko — входящие ссылки. Поисковая система присваивает ссылкам свой собственный Host Rank — он нагляднее, чем PageRank, так как в основе лежит линейная зависимость, а не вычисления по экспоненте. Другие полезные окончания поисковых запросов в blekko: /seo , /sitepages. К сожалению в системе недостаточно данных о неамериканских ресурсах.
    • верхние строчки соответствующих директорий Google и Яндекс. (Поиск сайтов может быть расширен за счет соседей по близким по смыслу уровням иерархии каталогов. Например, если сайт можно зарегистрировать в Computers > Internet > Web Design and Development > Designers, то весомые ссылки на него можно получить и в других разделах Computers > Internet > Web Design and Development, а также в каталоге другой ветки — Business > Business Services > Design — ссылки по теме есть на каждом уровне)
    Результаты стоит записать в таблицу со столбцами Google inlinks, SEMrush links, SEMrush linkdomain, тИЦ, PR (кстати, этот показатель можно снять для каждой страницы сайта); каждая строка — сайт-конкурент.
  4. Кроме всего прочего надо определиться с количеством целевых страниц. В большинстве случаев следует оптимизировать одну страницу под одно ключевое слово/фразу. Но изредка лучшей стратегией будет оптимизация одной страницы под все темы. Разрабатывая стратегию оптимизации, следует пресекать ссылочный каннибализм — то есть оптимизации нескольких страниц под одно и тоже ключевое слово/фразу. Для этого надо следовать трем правилам:
    • Избегать дубликатов содержания, при необходимости отмечать оригинальную страницу <link rel="canonical" href="http://www.example.com/optimized-page.htm" />
    • Оптимизировать под ВЧ страницы верхних уровней.
    • Писать уникальные мета-теги description и заголовоки браузера title.
  5. Прежде, чем переходить к следующему этапу, желательно провести экспериментальную кампанию контекстной рекламы. Объявления, появляющиеся в течение нескольких недель в результатах поиска по запросам из семантического ядра, покажут, на сколько посещений можно рассчитывать, продолжив SEO-практику. По итогам кампании определяется окончательный бюджет оптимизации, учитывающий стоимость привлечения каждого пользователя.

Повышение релевантности

Выбираются целевые страницы — те, через которые планируется получать трафик с поисковых систем. Как правило это первая страница и 3-20 внутренних: титульных страниц разделов, страниц основных продуктов, других целевых страниц. В тексте каждой из них повышается плотность одного из слов/фраз, входящих в семантическое ядро. Одновременно оптимизируется информационная и ссылочная структура, HTML-код всего сайта, делается точная настройка сервера.

Форма. Уровень страницы. Обязательно

Следует оптимизировать каждую страницу по одному (максимум, двум) ключевым словам (фразам).

  1. Мета-тег description < 155 знаков. 1-2 релевантных ключевых слова/фразы.

    Meta-тег description — пишем так, чтобы выделяться и привлекать в результатах поиска

    1. Нерелевантные содержанию страницы ключевые слова здесь не нужны.
    2. Без оформительской пунктуации.
    3. Без превосходных степеней.
    4. Call-to-action
      • Купи один, второй — в подарок.
      • Скидка 10% на первый заказ.
      • Мы снизем ваши расходы на 15%.
    5. Фокус на потребностях пользователя, а не на товаре.
      • Плохо: хорошии батареи.
      • Хорошо: в вашем доме будет теплее.
    6. Если рынок сбыта локальный — указать: «московская студия веб-дизайна».
    7. Краткое изложение опыта:
      • Создание сайтов с 1999.
      • Свыше 500 клиентов.
  2. Заголовок браузера title < 70 знаков, включая пробелы — 7-10 слов.
    Наименование страницы | Наименование раздела | Название компании, род деятельности.
    Например:
    Корпоративные СМИ | Услуги | «Полилог»: марктеинг, PR (пиар) еtс.
    Как минимум, в title, H1 и одном из H2 должно использоваться одно и то же ключевое слово/фраза в одной и той же форме (падеж, склонение).
  3. Description: 150-200 знаков. (Проверить эффективность готовых title и мета-тегов можно с помощью специальных веб-служб Или веб-сервис (англ. web service) — программа, которая запускается и работает в браузере.). Бесполезно включать в каждый title и description длинный список повторяющихся от страницы к странице ключевых слов.
  4. Уникальный заголовок H1.
    • Заголовок первого уровня может быть идентичным тегу title и URL или немного отличаться. Кроме SEO-функции должен выполнять свою прямую обязанность: привлекать внимание и сообщать суть материала. Это достигается с помощью:
      • Любопытства.
        Необычное сочетание слов, оксюмороны (SEO — это диетическая кола). Незаконченная мысль, вопрос. Конфликт. Отрицание широкораспространенного мнения.
      • Личной выгоды читателя. Как сделать… 10 способов… Простой способ… Введение в… Бесплатный…
      • Сильных эмоций. Убийца, трагический, поразительный, ошибки, отвал башки, необычный, эпично, бесконтрольный, невероятный, неожиданный, примечательный.
    • Максимальный эффект достигается, когда подзаголовки с ключевыми словами используются, как ссылки на внутренние страницы. Злоупотреблять не стоит.
  5. Минимум 80 слов уникального содержания.

Форма. Уровень страницы. Желательно

  1. Одно-два ключевых слов (фраз) в заголовках H2 и H3 (эффект использования ключевых слов в заголовках низших уровней — H4, H5, H6 — не выше, чем от использования в обычном тексте).
  2. Одно-два ключевых слов (фраз) в URL. URL, а также названия графических файлов должны быть описательными, при этом максимально полно использующими ключевой лексикон. В них предпочтительно использовать дефис вместо нижнего подчеркивания — чтобы поисковики могли находить документ не только по точной фразе, но и по отдельным словам. Например: /btl-promotion-technics.htm предпочтительней, чем /btl_promotion_technics.htm. (Вместе с тем, если не удастся полностью избавиться от атрибутов id, в них следует использовать нижнее подчеркивание — во избежание потенциальных ошибок программных сценариев)
  3. Минимум одна ссылка с ключевым словом/фразой.
  4. Три-четыре ключевых слова (фразы) в основном тексте. И хотя бы одно из клбючевых слов/фраз — жирным шрифтом.
  5. Минимум одна картинка.
    • Одно-два ключевого слова (фразе) в alt-атрибуте.
    • Хотя бы одна из сторон не меньше 300px.
    • Подклишевки у картинок.
  6. В ссылках на первую страницу должно использоваться ключевое слово/фраза, под которую первая страница оптимизирована (слово/фраза используемое в title, H1, H2).
  7. Для слайд-шоу — расшифровка картинок.
  8. Для видео — расшифровка всего ролика.
  9. Для видео — слово «видео» должно быть включено в тег title и мета-тег description.

Форма. Уровень страницы. Желательно (продолжение)

  1. Семантическая разметка в формате микроданных для расширенных описаний по стандартам schema.org
  2. Мета-тег keywords — влияние на результаты поиска почти не оказывает, можно отказаться.
  3. Мета-теги Global GeoURL. Для Москвы:
    <meta name="geo.position" content="55.755786;37.617633" />
    <meta name="ICBM" content="55.755786,37.617633" />
    Генератор GeoURL
  4. Как можно меньше исходящих ссылок на далекие по тематике ресурсы. Лишние с точки зрения основной темы ссылки не показываются поисковым машинам:
    <a href="/" rel="nofollow"></a>.
    Это относится и к внутренним страницам. Например, политике конфиденциальности, инструкциям по включению JavaScript в браузере, или формам отзыва на продукт — страница фактически одна и таже, а URL разные, с точки зрения поисковых систем это плохо (для существующих сайтов в данном случае лучше nofollow в ссылках лучше заменить meta-noindex непосредственнон а страницах).
  5. Вместе с тем — по одной-две исходящих ссылок на лидеров SERP — на странице. В этих ссылках важно использовать ключевые слова, а размещать их — поближе к «подвалу» (чтобы посетитель не ушел раньше времени). Например, так, как это делают составители Википедии.
  6. Cледует избегать параметров URL (англ. query strings) — обычно начинаются со знаков вопроса и амперсандов: ? и &.
    (вроде http://polylog.ru/ru/news_folder.php?docId=3342&PHPSESSID=1859
    aface13e584f385933a41a0da48b    ).
  7. На каждой странице оставить контактную информацию: телефон и почтовый адрес — это повысит значимость страницы по запросам земляков. (Телефон желательно выделять tel-ссылкой, что пользователи мобильных устройств могли запускать автоматический набор номера, щелкнув по ней. Например, <a href="tel:+7-916-123-4567">8 (916) 123-45-67</a>. В такую ссылку можно заключать только номер, текст вроде «позвони мне» не допускается; в атрибуте href номер должен соответстовать международным стандартам записи, в тексте ссылки допускается произвольный формат).
  8. Дополнительные указания на связь текущей страницы с другими страницами сайта с помощью элементов base и link.
    • <link rel="help" href="http://www.polylog.ru/marketing-communications-community/pr-btl-help/index.htm" />
    • <link rel="home" href="http://www.polylog.ru/" />
    • <link rel="index" href="http://www.polylog.ru/find/sitemap.htm" />
  9. Люди не любят слишком длинные страницы, а поисковики — слишком короткие. Следует придерживаться золотой середины: очень приблизительный среднеарифметический размер — 760x1000 пикселей (от 200 до 500 слов — грубо говоря, от половины до целой стандартной word-овской страницы); на него следует ориентироваться при верстке текстов и иллюстраций.
  10. Важно минимизировать файловый размер страниц — чем он меньше, тем лучше. Точных данных нет, но считается, что около 50% пользователей подключены к интернету по устаревшим каналам связи, пропускная способность которых не более 56 Кбит/с.
  11. Для запрета индексирования служебных участков текста Яндексом можно использовать комментраий <noindex>.
    <!--noindex-->текст, индексирование которого нужно запретить<!--/noindex-->

Форма. Уровень сайта. Обязательно

  1. Правильно составленный robots.txt Невидимый посетителям файл, который указывает поисковым роботам, что просматривать, а где нет. — запрещающий поисковым системам индексацию страниц, не соответствующих основной тематике сайта (лучше использовать директиву noindex, а не более известную disallow).
  2. Карта сайта Страница ссылок на все значимые страницы сайта. и титульные страницы каждого раздела и подраздела должны содержать ключевые слова и внутренние ссылки. На каждой такой странице (включая карту) должно быть не более 100 ссылок. Если не получается, лучше сделать многостраничные карты, чем превысить указанный лимит.
  3. Google sitemap Файл, специально созданный для поисковой системы Google и содержащий ссылки на все значимые страницы сайта. (GSiteCrawler — программа создания sitemap; эту же карту следует добавить и в Bing). Влияние на результаты поиска слабое.
  4. Редирект (англ. redirect — переориентировать) — принудительный, не запрошенный пользователем переход на другой адрес. 301 (301 Permanent Redirect). Для машин «однокоренные» адреса с www и без www не являются синонимами: для них polylog.ru и www.polylog.ru — принципиально разные URL. Поэтому для повышения своих показателей в поисковой выдаче, следует установить единственный, самый главный адрес ресурса — лучше с приставкой www, так привычнее. Для этого в настроечных файлах добавляется запись о редиректе 301:
    Apache (файл .htaccess)
    Редирект на новый домен:
    Redirect 301 / http://www.new-example.com/
    Редирект директорий:
    RewriteRule ^oldfolder/(.*) http://www.example.com/newfolder/$1 [L,R=301]
    Редирект файлов:
    RewriteRule ^old-folder/old-file.htm$ http://www.example.com/new-folder/new-file.htm [R=301,L]
    Кроме того, 301-й редирект используют:
    • При переносе сайта на новый домен;
    • При слиянии двух сайтов.
    Для страниц временно меняющих адрес следует использовать редирект 302.
    Redirect 302 /news/2010/index.htm http://www.example.com/news/index.htm

Содержание

  1. Мониторинг интересов публики для использования популярных ключевых слов в собственных новостях.
  2. Новости нужны частые и регулярные, пусть и далеко не сенсационные — поисковые машины им «симпатизируют».
  4. После создания корпоративного блога:
    • В первую очередь, разумеется, сделать его интересным для читателя (см. советы по редакционной политике и контент-маркетингу).
      • Обновлять не реже раза в день (особенно стараться в дни пиковых посещений, когда посетители начинают заходить с какого-нибудь популярного сайта, поместившего ссылку на блог)
      • Создавать тематические и периодические (лучшее за месяц, за год) обзорные материалы с сылками на статьи из архива.
      • Создать блок «Самые популярные заметки» и, возможно, «Самые популярные комментарии».
      • При каждом удобном случае ссылаться из свежих заметок на архивные.
      • Добавить блок ссылок "Также по теме".
      • Формы комментариев и рекламные материалы следует вводить только после того, как блог станет популярным (10 подписок на RSS, 20 посетителей в день). Отсутствие комментариев и навязчивая реклама не способствует укреплению репутации.
      • Просить широко известных в нише маркетинговых коммуникаций авторов, чтобы они и здесь написали заметку-другую или рецензировали чужую.
      • С ростом популярности дневника привлекать к работе над ним начинающих и перспективных блогеров.
      • Вести еженедельный обзор близких по теме блогов, предварительно договорившись с авторами об обмене ссылками.
    • Добавить блог и RSS-поток (оптимальное ретраслируемый через FeedBurner) в специализированные каталоги и трансляторы. Но лучше заказать регистрацию у частных регистраторов веб-ресурсов: в разеделе «Регистрация в каталогах» форума searchengines.ru надо поискать «ветки» со словом «RSS»).
    • «Пинговать (англ. ping — свистеть) настойчиво навязывать какому-либо интернет-ресурсу информацию, непрерывно посылая ему небольшие порции данных.» системы поиска по блогам — сообщать о важных обновлениях дневника (FeedBurner это делает).
    • Публиковать и дать возможность публиковать другим ссылки на свои заметки в соц-медиад. Для этого создать блок соответствующих кнопок.
      Однако здесь следует не переусердствовать, чтобы не создать себе репутацию пустобреха. Широкой огласки заслуживают только «гвозди номера», передовицы — приблизительно 5% от общей массы текущей информации (это ограничение, конечно, условно и формально, но полезно). Кроме того, при публикации ссылки в англоязычной службе следует составлять названия и описания на английском.
    • Ссылаться на близкие по тематике блоги (в надежде на взаимность) — искать их можно на Technorati и соответствующей службе Яндекса, сортируя затем по количеству входящих ссылок.
    • С каждой заметки блога, с каждой страницы других разделов, тем или иным образом затрагивающих тему одного из продуктов/услуг компании, должна быть CTA-ссылка на целевую страницу этого продукта или услуги — одну, а не несколько разных. Эта целевая страница в свою очередь должна содержать CTA дальнейшего действия (регистрации, загрузки, покупки, заказа консультанта), а также четкое определение товара — с использованием ключевых слов.
  5. Еще RSS (помимо ленты блога, это может быть общая лента, лента форума и т.д.). На первой странице следует объявить подписку на все существующие ленты (появляются выпадающим меню при нажатии кнопки rss в адресной строке):
    <link rel="alternate" type="application/rss+xml" title="Все обновления сайта" href="http://www.example.com/rss/all/" />
    <link rel="alternate" type="application/rss+xml" title="Блог" href="http://www.example.com/rss/blog/" />
    <link rel="alternate" type="application/rss+xml" title="Форум" href="http://www.example.com/rss/forum/" />
    <link rel="alternate" type="application/rss+xml" title="Новости" href="http://www.example.com/rss/news" />
  6. Email-рассылка (форма и процесс; как минимум — для блога). В каждом письме (newsletter) — по одной 1-2 ссылки на сайт.

Повышение «ссылочной массы» (Link building)

Чем больше ссылок на сайт, чем больше суммарный «вес», тем лучше — во-первых, больше прямой трафик, во-вторых, лучше позиции в поисковых результатах. Следует прилагать усилия в двух направлениях:

  1. естественные ссылки с авторитетных источников (качество содержания и/или продукта плюс личное общение с авторами)
  2. большое количество ссылок с любых более-менее релевантных ресурсов («мусорных ссылок»).

Cсылки нужно собирать не только на первую, но и на внутренние страницы (считается, что под высокочастотные (ВЧ) запросы следует оптимизировать первую, под НЧ НизкоЧастотные запросы — слово из сленга оптимизаторов, означает непопулярные поисковые запросы. Аналогична этимология терминов ВЧ и СЧ — внутренние). Чем выше отношение ссылок на внутренние страницы (a) к общему количеству ссылок на сайт (b) — коэффициент а / b х 100% — тем лучше

Количество ссылок, необходимых для продвижения на первые страницы поисковой выдачи, зависит от агрессивности конкурентной среды. Но для ориентира можно взять план в 50-150 новых ссылок в день (от тысячи до 3 тысяч в месяц). Впрочем, для достижения гарантированого результата, следует ориентироваться на аналогичный показатель у конкурентов (см. аналитический этап).

Еще одним ориентиром могут служить добытые оптимизаторами опытным путем приблизительные коэффициенты влияния, оказываемого на страницу входящими ссылками.

В уме надо также держать поверье, что на значимость ссылок для поисковых систем (особенно для «Яндекса») зависит от уровня вложенности (УВ): с первой страницы и с титульных страниц подразделов (УВ2) ссылки ценятся гораздо выше, чем со страниц более низких уровней вложенности (УВ3+). Причем, как правило, ссылки с главных при прочих равных показателях стоят дороже ссылок УВ2. Вывод: при всех прочих равных показателях лучшее соотношение цена/качество у ссылок УВ2. Вышесказанное не относится к каталогу «Яндекса»: для повышения тИЦ и позиций сайта в каталоге подойдут страницы-доноры любого уровня вложенности — даже дешевые УВ3+. Впрочем, покупка ссылок, не оказывающих заметное влюяние на ссылочное ранжирование, представляется бессмысленной.

  8. Купить на чужих сайтах площади под собственные рекламные статьи.

    • intitle:регистрация+сайта "пиар"
      intitle:add+site "public relations"
    • intitle:добавьте+свой+сайт "пиар"

    • intitle:каталог "пиар"
      intitle:directory "public relations"
    • intitle:добавить+статью "пиар"
      intitle:add+article "public relations"
    • intitle:разместить+статью "пиар"
      intitle:submit+article "public relations"
    • intitle:каталог+статей "пиар"
      intitle:directory+article "public relations"
    • intitle:размещение+статей "пиар"

    • intitle:сайты "пиар"

    • Yahoo Local

  4. Social Media Marketing (SMM). Речь идет не о спаме в блогах, форумах и личных страницах популярных пользователей (хотя и этим можно заняться, если совесть позволяет). «Честный» SMM основан на добавлении сайта и отдельных заметок в службы социальных закладок и новостей, а также в специально созданные в популярных соц. сетях учетные записи. Паралелльный процесс «честного» SMM — повышение собственного авторитета в выбранных службах. Алгоритм следующий.
    1. В крупнейших соц-медиа зарегистрировать пользователя, в псевдониме и «настоящем имени» которого использовалось бы название продвигаемого сайта: например, polylog и Полилог Пиар.
    2. Проявлять социальную активность. Опубликовать, например, на каждом из выбранных социальных узлов свои интернет-закладки. Причем метить как можно большим количеством тегов — это принесет обратные ссылки со страниц тегов.
    3. Найти на каждом сайте тематически близкие продвигаемому ресурсу кружки по интересам — пользовательские группы. Например, группу «Русский пиар», группу «Пиар vs PR», группу «Случайные связи с общественностью». Зарегистрироваться в них, получив таким образом тематические обратные ссылки на пользовательскую страницу.
    4. Найти самых популярных пользователей каждого сайта (особенно тех, чьи интересы совпадают с тематикой продвигаемого ресурса). Вступить в те группы, в которых они участвуют, публиковать от своего имени их интернет-закладки и т. п.
    5. Найти на страницах тематических категорий самые популярные интернет-закладки, добавить их в свою коллекцию.
    6. В течение хотя бы недели-двух следует ежедневно добавлять что-нибудь новое на свою пользовательскую страницу.
    Подробнее см. здесь и здесь (продвижение сайтов с помощью StumbleUpon).
  5. Добавление сайта в галереи веб-дизайна (только после публикации англоязычной версии — так как большинство таких галерей нерусские).
  6. Если есть вакансии: публикация объявлений о приеме на работу на сайтах университетов, институтов, специализированных сайтах вакансий.
  7. Мониторинг обсуждений .

Экстенсивные, трудоемкие командные методы

  1. «Гостевые посты», ответы на вопросы по релевантной теме, комментарии отраслевых новостей со ссылками на автора. Для этого нужно с помощью Google Alerts, Twilert и Hyper Alerts проводить постоянный мониторинг:
    • Упоминаний названия компании, конкурентов, лидеров отрасли.
    • Гостевых постов конкурентов.
    • Запросов гостевых постов.
    • Новых отраслевых блогов (можно предлагать гостевые посты).
    • Вопросов по отраслевой теме.
    • Отраслевых новостей (можно сделать комментарий или важное уточнение за ссылку).
    Если источники достойные, предлагать долгосрочное сотрудничество — обмен ссылками через форму обзоров, например.
    Чтобы облегчить себе процесс поиска площадок, нужно предоставлять свой блог под чужие заметки.
    • Завести на каждой странице своего блога CTA-блок «Хотит стать автором нашего блога?», который должен вести на страницу с правилами внештатных авторов. Пример
    • Присланные материалы тщательно отбирать и редактировать
    • После того, как заметка опубликована, отрекламировать ее в своих аккаунтах в соц-медиа. Подписаться на автора в Твиттер и Google+, ответить на пару его твитов, что-нибудь прокомментировать в его блоге.
    • После этого написать коротенькое письмо с предложением опубликовать свою заметку в его блоге.
    • В своей краткой автобиографии в конце заметки делать ссылку на свой профайл на Google+ с атрибутом rel=author.
      • Атрибут указывается непосредственно в URL:
        <a href="https://plus.google.com/105851932814207462551?rel=author"></a>
      • Затем надо авторизовать в Google, зайти на свою страницу.
      • Нажать синюю кнопку Edit Profile в правом верхнем углу
      • Нажать блок Contributor to в правой колонке.
      • Нажать Add custom link.
      • Добавить ссылку на свой сайт.
  2. Написание или «перепечатка» собственных заметок и их обсуждение в местах массового скопления народа:
    • На сайтах отраслевых новостей (ищем).
    • В отраслевых форумах и блогах (интересуют только те сайты, которые не помечают ссылки в комментариях атрибутом rel="nofollow"; нужные ссылки размещаются везде, начиная с автобиографической страницы, с автографа, который будет появляться рядом с репликой; в автографе следует указать сферу деятельности: «маркетинговые коммуникации — пиар, BTL и пр.»);
    • В крупных центрах обмена мнениями — Google Groups и Yahoo! Answers (в России — Ответы@Mail.Ru)
    • В сетевом самиздате и в сборниках откровенно рекламного типа. Реклама в самодеятельных газетах, разумеется, должна быть скрытой — в экспертных статьях, обзорах, советах. Найти специализированные СМИ (в том числе и для русского рынка) можно в поиске по новостям Yahoo! — по запросу с ключевыми словами; и с помощью подборки Alexa>Related links. Во избежание санкций со стороны поисковых систем, пресс-релизы должны отличаться друг от друга: во-первых, те, что публикуются на собственном сайте, должны оставаться в единственном экземпляре; во-вторых, в материалах для самиздата тоже должно быть хоть какое-нибудь разнообразие — грубо говоря, одна уникальная заметка на каждые 15-20 сайтов.

    Если позволит бюджет, можно нанять внештатного сотрудника для рассылки заметок по редакциям.

  3. Обмен ссылками. Эта некогда самая популярная у оптимизаторов методика требует приложения немалых усилий. Однако в условиях ни на минуту не прекращающейся борьбы поисковых систем с платными ссылками сохраняет свою актуальность для SEO-процесса. Тем более, что, как мнимум, часть работы можно автоматизировать.
    Вставлять ссылки нужно в тексты «Энциклопедии» (идеально подходят на роль так называемого «гиперконтекста»):
    • В ссылки следует добавлять ключевые слова (или окружать ссылки ключевыми словами) — так чтобы сохранялся смысл абзаца; подходящий формат — «комментарий эксперта». В статье можно размещать множество ссылок на разные источники (один абзац — одна ссылка). Для наиболее популярных партнеров отводить под ссылку по отдельной (коротенькой) заметке. Ссылки нужно размещать только на паритетных условиях.
    • Внутри статей следует вставлять пару ссылок на собственные страницы.
    • Количество исходящих ссылок на одной странице не должно превышать 10.
    • Ни в адресе страниц, ни в их заголовках (как title, так и h1-h6), ни в тексте ссылок на эти страницы не должны упоминаться их подлинный характер: обмен ссылками, link exchange.

    • Меняться следует с сайтами, которые ссылаются на лидеров SERP по интересующему запросу — прямой (не более 10% от общего количества ссылок) и перекрестный, 3-сторонний. Количество, потенциальных «доноров», вычисляем на основе конкурентного анализа (в самом общем случае — это 20-40 страниц).

      Предложение об обмене должно быть справедливым. Наиболее авторитетным ресурсам следует предлагать наиболее выгодные условия размещения их ссылок. В случае отказа от бартерных отношений, можно предложить хорошую цену.

      С другой стороны, если инициатива обмена исходит извне, надо быть избирательным: нельзя обмениваться ссылками с первым встречным-поперечным только потому, что «он сам предложил». Сначала нужно проверить «вышедший на связь» сайт — зайти, поискать в Google, посмотреть PageRank (если меньше 3, то обмен не имеет смысла), проверить, не обманывает ли веб-мастер своих контрагентов (замена ссылок, маскировка В реальности — ссылка одна, а в статусной строке браузера с помощью JS указывается другая. с помощью JavaScript, запрет индексации Проверить robots.txt можно с помощью SearchStatus — см. ниже. страниц с исходящими ссылками, сокрытие его реального PageRank или запрещенной индексации Нечестные веб-мастера могут поместить каталог во фрейме, чтобы спрятать его реальный адрес, и, в конечном итоге, запретить его индексацию в robots.txt. с помощью фреймов, отсутствие ссылок на каталог с других страниц сайта, клоакинг (англ. cloaking) — поисковому роботу показывается оптимизированный сайт, посетителям — удобочитаемый.).

      И еще раз: меняться нужно только с ресурсами на схожую тематику. И каждый месяц проверять свои входящие ссылки.

      Собственные ссылки должны быть:

      • разнообразны
      • оформлены, как вырванные из контекста фразы
      • вести не только на первую, но и на важные внутренние страницы
      • ключевые слова не обязательно должны помещаться в текст ссылки — как минимум, в половине вариантов они должны быть просто поблизости от ссылки (не далее двух слов)
      • оставлять важным или обладающим хорошим слогом партнерам возможность самим составлять или редактировать текст ссылки, предлагаемой им для обмена.

      Будет замечательно, если и контрагентов удастся убедить следовать тем же принципам.

  4. Полезен также обмен ссылками с каталогами профессиональных объединений, сайтами СМИ, некоммерческих, и, если возможно, государственных организаций.
    Для поиска англоязычных некоммерческих ресурсов используется функция выбора домена первого уровня в настройках поиска Yahoo.
  5. Публикация на «досках» бесплатных объявлений, лучше через регистраторов (в разеделе «Регистрация в каталогах» форума searchengines.ru надо поискать «ветки» со словами «объявлений»).
  6. Благодарности и рецензии (в надежде на взаимность):
    • на товары (книжки) и услуги, появляющиеся в целевом секторе рынка — в Amazon (в том числе — в собственных списках желанных покупок), ePinions и других крупных магазинах и товарных базах
    • на сходные по тематике сайты — в Alex-е и аналогичных каталогах
    Метод — условно бесплатный. За право написать рецензию, возможно, придется что-нибудь купить.
  7. Статьи о коллегах, на сайтах которых есть рубрика «Пресса о нас» (In the News; запрос в Google) в обмен на ссылки.

Можно, если осторожно

  1. Покупка заметок блоггеров: Блогун, Social Media. Эффект трудно прогнозируемый и замедленного действия (проявляется с задержкой до двух месяцев). Следует проявлять осторожность: расходуя по $50 в месяц, покупать заметки небольшими партиями (разумеется, в блогах с хорошими показателями тИЦ и PR и, желательно, редко обновляемых, чтобы ссылка подольше оставалась на первой странице). После каждой покупки надо планировать будущие приобретения: прекращая тратиться на материалы в тех онлайн-дневниках, которые, может, и добавляют какую-никакую «авторитетность» нашему ресурсу, но не приносят прямой трафик (хотя бы 20-100 посетителей).
  2. Имитация хакерского взлома. На пару-тройку суток меняем содержание главной страницы сайта, так, как будто над ней поглумились хакеры (можно с фотографией голой женщины или с каким-нибудь актуальным политическим подтекстом — например, модальное окно с пиктограммой опасности (желтый треугольник с восклицательным знаком), вопросом «Спорим, что Медведева не переизберут на второй срок?» и кнопками Yes, No, Cancel). Сообщаем о вандализме в базы дефейсов (англ. deface — портить, искажать) — тип хакерской атаки, подмена веб-страницы: злоумышленники могут опубликовать на сайте жертвы рекламу, угрозы и т. п. — пишем взволнованное письмо, а не пресс-релиз. Помянутые базы ссылаются на «взломанный» сайт. Через день-другой, восстанавливаем главную страницу, но ссылки на сайт в базах дефейсов остаются.
  3. XSS-атаки. XSS расшифровывается как Cross Site Scripting (термин, который лучше не переводить) и означает использование слабых мест в защите сервера для редактирования чужой HTML-страницы (добавления текста, ссылок, картинок etc). Делается это так:
    1. Ищем формы поиска на подходящих сайтах. (Например, в Yahoo с помощью команды site:.gov search находим формы поиска на сайтах государственных учреждений, в первую очередь — американских).
    2. Отбираем уязвимые сайты — c помощью программы Site-Auditor и панелей инструментов для Firefox: Web Developer и Search Status. Нас интересуют сайты:
      • без проверочных полей ввода
      • не скрывающие от поисковых систем формы (с помощью robots.txt, мета-тега robots)
      • не изолирующие формы (то есть ссылающиеся на них с других своих страниц)
      • не содержащие в URL HTML-код
      • не обесценивающие исходящие ссылки с помощью атрибута rel="nofollow".
    3. С помощью все того же Web Developer меняем метод отправки формы с POST на GET: Forms > Convert form methods.
    4. Вставляем нужный код, отправляем запрос — в итоге код публикуется на странице результатов поиска по сайту. Примеры.
