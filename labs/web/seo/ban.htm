Если возникли подозрения, что сайт исключили из баз поисковых систем

Яндекс

Проверяем опасения:

  • Яндекс.Вебмастер: Проверить сайт;
  • Yaback - сервис проверки обратных ссылок.

Если сайт в базе не находим, пишем письмо, с вопросом «в чем дело?». Если отвечают, что сайт забанен (англ. ban — запрещать) — наказание, которое поисковые системы налагают на нарушителей их правил — запрет индексации сайта и удаление его из базы., то ищем в логах сервера вот такую строку sb.yandex.ru/verify.xml.

Именно с этого адреса модераторы проверяют сайты. Обращаем внимание на все страницы, которые были просмотрены с этого адреса. Пытаемся понять, что именно не понравилось «Яндексу» на этих страницах. Исправляем. После этого пишем новое письмо в «Яндекс»: мол, причина, нарушавшая п. 3.5 Лицензии устранена. Если все верно сделали, приходит письмо, что окончательно вопрос о снятии санкций с сайта будет решаться примерно месяц.

Google

Проверяем опасения следующими поисковыми запросами.

  • site:www.polylog.ru — есть ли сайт в индексе;
  • cache:www.polylog.ru — был ли сайт в индексе;
  • link:www.polylog.ru — есть ли в индексе ссылки на сайт.

В случае бана, списываемся с модераторами поисковых систем.

Примерно тот же алгоритм по исправлению причин бана.

Исправив страницы-нарушители, заполняем форму Reinclusion request (или Google Webmasters Tools > список Tools в правой колонке > Request reconsideration).
